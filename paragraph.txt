I'm particularly happy with how I handled the retrieval side. Since the transcript is in
English but queries can come in Czech, I added a translation step before hitting the vector
database — without that, the semantic similarity scores are noticeably worse. For comparison
questions (Victorian vs. Edwardian), the LLM splits the query into two separate searches and
I merge the results, which made the difference between a mediocre and a solid answer on that
scenario.

I also built in a model cascade so if one Gemini model hits its free-tier quota limit, the
service automatically tries the next one — no crashes, no manual restarts.

On top of that I put together a proper Web UI with a dark glassmorphism design and custom CSS,
which I think makes the project feel much more complete than a bare API.

The one thing I didn't get to is streaming, but everything else works end-to-end.
