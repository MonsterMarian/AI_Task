AI Programmer - Junior/Mid Level
Documentary Q&A Backend
Overview
Your task is to build a backend service that allows a user to ask natural language questions
about a long documentary, and receive accurate answers with references back to the source
material.

The documentary is provided to you as a transcript text file. You do not need to process video or
audio.

Context: This type of system ÔÇö retrieving relevant information from a large corpus and
answering questions grounded in that source ÔÇö is directly representative of the work you
will do on this team, including building tools over large codebases and technical
documentation.

´éÀ A plain text transcript file of the documentary
´Ç¡ "An Investigation into Domestic Dangers Through History"
´Ç¡ Approximately 190KB of plain text ÔÇö large enough that it fits in modern LLM context
windows, but you are explicitly required to use a retrieval-based approach regardless

´éÀ This assignment document
Deliverable
A working backend application that exposes an HTTP API with the following behaviour:

Field Description
Endpoint POST /ask
Request body { "question": "<user question>" }
Response body { "answer": "...", "sources": [ { "timestamp": T, "excerpt": "..." } ] }
Sources At least 2ÔÇô3 ranked source references per answer
Requirements
Functional
´éÀ The system must use a retrieval-based approach: embed the transcript, store in a vector
index, and retrieve relevant chunks per query
´Ç¡ Feeding the full transcript directly into the LLM context on each request is not
acceptable, even though it would technically fit
´Ç¡ Your design doc must describe your chunking and retrieval strategy ÔÇö this is a core

´éÀ Questions must be answered in natural language based solely on the transcript content
´éÀ Each answer must include source references with rough time code and a short excerpt
´éÀ Sources must be ranked by relevance to the question
´éÀ The service must start and run with a single command (e.g. docker compose up or a
simple readme instruction)
Non-functional
´éÀ No mandatory cloud dependency ÔÇö the service must run without requiring a paid
subscription
´Ç¡ Free-tier APIs (Groq, Gemini, etc.) are acceptable
´Ç¡ A fully local setup (e.g. Ollama) is acceptable and noted as a bonus ÔÇö see below
´Ç¡ If you use a free-tier API, document clearly how to obtain and configure a key
´éÀ The LLM backend should be configurable ÔÇö ideally switchable between providers

´éÀ Response time under 30 seconds for a typical question on standard hardware
Required: Short Design Document
Include a DESIGN.md file in your repository. It does not need to be long ÔÇö half a page to one

´éÀ A question about a named person or location
´éÀ A vague or broadly phrased question
´éÀ A question whose answer is not in the transcript at all

´éÀ Answer correctness and grounding in the source material
